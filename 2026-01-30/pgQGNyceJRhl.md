### 改进建议

1. **补充核心组件关联**  
   在 EventLoop 部分增加以下内容：  
   ```markdown
   - **Channel**：网络连接载体，通过 `ChannelPipeline` 处理数据流。
   - **Handler**：事件处理器，如 `ChannelInboundHandler` 处理入站数据。
   - **Pipeline**：事件处理链，串联多个 Handler 实现业务逻辑。
   ```

2. **增加实战配置指南**  
   在 EventLoopGroup 部分添加选型矩阵：  
   ```markdown
   | 场景          | 推荐方案               | 理由                                                                 |
   | ------------- | ---------------------- | -------------------------------------------------------------------- |
   | 高并发 RPC    | `NioEventLoopGroup`    | 默认适配多核 CPU，线程池自动扩缩容。                                 |
   | Linux 高性能  | `EpollEventLoopGroup`  | EPOLL 事件模型比 NIO 更高效。                                       |
   | 测试环境      | `DefaultEventLoopGroup`| 单线程模式，便于调试。                                               |
   ```

3. **补充线程模型说明**  
   ```markdown
   - **单线程 EventLoop**：适用于 I/O 密集型场景（如长连接），避免线程切换开销。
   - **多线程 EventLoop**：适用于 CPU 密集型场景（如业务计算），通过线程池分担负载。
   ```
